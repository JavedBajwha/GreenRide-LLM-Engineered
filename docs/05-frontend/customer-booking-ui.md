# Customer Booking UI

## Purpose

Defines the exact UI behaviour and structure for the customer booking experience.

---

## Core Screens

### 1. Search & Quote
- pickup input
- dropoff input
- date/time selector
- passenger/luggage selector
- search button

### 2. Quote Results
- vehicle cards
- pricing display
- estimated time
- selection action

### 3. Booking Form
- customer details
- extras
- notes

### 4. Payment
- payment method selection
- confirmation

### 5. Confirmation
- booking summary
- status

---

## UX Rules

- fast interaction
- minimal steps
- mobile-first layout
- clear call-to-action

---

## Summary

Customer booking UI must be simple, fast, and reliable.

It is the primary revenue entry point of the platform.
