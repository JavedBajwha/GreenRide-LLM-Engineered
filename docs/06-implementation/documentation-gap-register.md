# Documentation Gap Register

## Purpose

This file records all known documentation gaps that could cause AI build drift, unsafe assumptions, or implementation inconsistency.

The goal is not to list every possible idea.
The goal is to capture missing or weak documentation that blocks safe AI-assisted implementation.

---

## Gap Categories

| Category | Meaning |
| --- | --- |
| missing | No dedicated contract exists |
| weak | Document exists but is too high-level for implementation |
| fragmented | Required information exists, but is spread across too many docs |
| unlinked | Correct docs exist, but their relationship is unclear |

## Rule

A gap is only resolved when:

1. the relevant document exists or has been upgraded
2. the document is linked from the right control/navigation docs
3. the master tracker status is updated
