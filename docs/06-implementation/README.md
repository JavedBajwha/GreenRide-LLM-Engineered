## Suggested Read Order For Documentation Program

When working on documentation completion, read in this order:

1. `documentation-master-tracker.md`
2. `documentation-gap-register.md`
3. `documentation-roadmap.md`
4. `required-documentation-coverage.md`
5. `session-continuity-notes.md`
6. `documentation-contract-template.md`

This order helps agents understand:
- what exists
- what is missing
- what phase is active
- what coverage is still incomplete
- what decisions were already made
- how new documents must be written
