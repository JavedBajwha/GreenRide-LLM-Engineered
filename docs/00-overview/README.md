# Overview

This folder contains the control documents for the GreenRide repository.

These files explain:
- what the repo is for
- what stage the project is in
- how AI coding agents must behave
- where the important documents live
- when implementation is allowed to continue

## Read Order

1. `current-state.md`
2. `doc-map.md`
3. `working-rules.md`
4. `implementation-rules.md`
5. `ai-stop-rules.md`

## Purpose

The earlier GreenRide documentation set was strong for architecture and product understanding, but too loose for deterministic implementation.

This folder closes that gap by adding explicit operating rules.
